# Online Retail ML Project

Прогноз покупок в течении месяца и сегментация клиентов интернет‑магазина на основе транзакционных данных.

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/GirenokNikita/Online_retail/blob/main/Online_retail_dataset_clean_final.ipynb)

## Ключевые результаты

- **Сегментация RFM/ABC/Процент_выкупа:** топ‑сегмент — 16,33 % клиентов; клиенты с выкупом < 50 % — низкоприоритетные.
- **Логистическая регрессия:** ROC‑AUC ≈ 0,977, F1‑score = 0,885 — отлично подходит для приоритизации клиентов в маркетинговых кампаниях.
- **Линейная регрессия:** R² = 0,38 (розница), R² = 0,41 (опт) — приоритет задачи в ранжировании клиентов, а не в абсолютной точности прогноза суммы покупки.
- **Дашборд:** на основе итоговой таблицы с сегментацией и прогнозами построен интерактивный дашборд (Power BI). С его помощью маркетинговый отдел может определять географию и группы клиентов под конкретную задачу: достаточно выбрать нужные сегменты и гео — внизу отобразится таблица со списком целевых клиентов (с ID, страной, прогнозом покупки,прогнозом выручки и сегментами).

## Что внутри

- Очистка данных: разделение покупок и возвратов, обработка выбросов (IQR), нормализация,логические условия.
- RFM,ABC‑сегментация, расчёт доли выкупа, вычисление итоговых баллов для ранжирования приоритета клиентов.
- Обучение и оценка моделей: логистическая и линейная регрессии, сравнение метрик, анализ важности признаков.
- Объединение результатов сегментации и ML‑прогнозов в единую таблицу приоритетов клиентов.
- Подготовка данных для дашборда и визуализация ключевых метрик.

## Структура файлов и папок

- `Online_retail.csv.gz` — исходные транзакционные данные.
- `Online_retail_dataset_clean_final.ipynb` — полный пайплайн анализа и ML (с кодом, проверками и комментариями).
- `ML/` — папка с сохранёнными обученными моделями (`.pkl` и аналоги) для быстрого применения без повторного обучения.
- `Report/` — чистый Jupyter‑ноутбук без кода: только выводы, интерпретации и итоговые рекомендации (удобно для презентации).
  - В папке `Report/` также лежат все визуализации (графики, boxplot, таблицы сегментов) — их можно сразу вставлять в отчёты и презентации.
- `Dashboard/` — папка, содержащая:
  - файл дашборда (`.pbix` для Power BI) — готовая интерактивная панель для аналитики;
  - скриншот дашборда — для быстрой демонстрации интерфейса без запуска Power BI;
  - финальную расчётную таблицу — основу для построения дашборда (CSV/Excel), по которой считаются все показатели и KPI.

## Как использовать дашборд (Power BI)

Дашборд сделан так, чтобы маркетолог мог быстро собрать целевую аудиторию без помощи аналитика.

### Основные фильтры (слева/вверху)
- **Сегмент клиента (RFM/ABC):** выбирай, например, «Лучшие / A / Высокий» или «Спящие / C / Низкий», чтобы сразу сузить аудиторию под тип кампании.
- **Страна/регион:** выбирай одну страну или группу стран — дашборд покажет долю и вклад региона в прогноз выручки.
- **Прогноз покупки (лог. регрессия):** фильтр по вероятности (например, >70 %) — удобно для «горячих» рассылок.
- **Прогноз суммы (линейная регрессия):** можно отфильтровать по ожидаемой выручке (например, топ‑20 % по прогнозу) для VIP‑предложений.
- **Процент выкупа:** фильтр «низкий» сразу выделит проблемных клиентов.

### Что видно на экранах
- **Сводные карточки вверху:** количество клиентов, которые купят в следующие 30 дней (по прогнозу), общая прогнозная выручка.
- **Графики динамики:** доли клиентов по сегментам.
- **Таблица внизу:** список клиентов под выбранные фильтры. В строках: `ID_покупателя`, `Страна`, `Сегмент-RFM `, `Сегмент-ABC `, `Прогноз покупки (вероятность)`, `Прогноз выручки`, `Процент выкупа`. Эту таблицу можно выгрузить в Excel.

### Примеры типовых сценариев
- **«Горячая рассылка»:** фильтр: сегмент «Лучшие / A», вероятность покупки >70 %, процент_выкупа средний или высокий. Результат: список самых перспективных клиентов для персонального предложения.
- **«Реактивация спящих»:** фильтр: сегмент «Спящие» , но сегмент не «Потерянные». Результат: список для письма «Мы скучаем» с промокодом.
- **«Фокус на регионах»:** фильтр: с помощью дашборда можно выделить клиентов по конкретной стране и на основе их поведения адаптировать маркетинговую рассылку под культурные особенности региона.

### Как обновить данные в дашборде
1. Запусти ноутбук в Colab (или локально) с новыми данными.
2. Экспортируй итоговую таблицу приоритетов в CSV (в коде есть блок сохранения).
3. В Power BI замени источник данных на новый CSV (или обнови через кнопку «Обновить»).
4. Все графики и фильтры пересчитаются автоматически.
